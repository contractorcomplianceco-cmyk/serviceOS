Build a full-stack web application called ServiceConnect with RoseOS Intelligence.

ServiceConnect is a custom field-service operating system designed to replace BlueFolder and eventually replace QuickBooks for a plumbing/service company. It must feel like a real operational platform, not a landing page or demo page.

Core purpose:
One system for every job, every tech, every invoice, every inventory item, and every customer update.

Build the first working prototype with realistic seeded data. Do not connect real APIs yet, but structure the app so ServiceChannel, GPS, VoiceConnect, merchant processing, and accounting integrations can be added later.

TECH STACK
Use:
- React
- TypeScript
- Vite
- Tailwind CSS
- Local mock data or lightweight backend storage if available
- Responsive design for desktop dispatchers and mobile technicians

VISUAL STYLE
Make it polished, clean, operational, and easy to scan.
Do not make it look like a marketing website.
Use a professional field-service command center style.
Avoid yellow-heavy branding.
Use a modern Rose OS feel: clean white workspace, deep navy/charcoal structure, blue accents, green for completed, red for urgent, amber only for warnings.

PRIMARY USERS / ROLES
Create role-based views for:
- Administrator
- Scheduler
- Service Manager
- Technician
- Lead Technician
- Billing
- Bookkeeper
- Subcontractor

Permissions should affect what each role can see:
- Technicians see assigned jobs and limited customer/job info.
- Schedulers see intake, calendar, routing suggestions, and assignments.
- Billing sees billing queue, invoice drafts, AR, payments, and job cost.
- Admin sees everything.
- Sensitive employee fields and job-cost/accounting data should be permission-gated.
