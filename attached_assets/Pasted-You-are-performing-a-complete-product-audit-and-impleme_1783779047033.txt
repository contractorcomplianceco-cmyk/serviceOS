You are performing a complete product audit and implementation-completion pass for the existing ServiceConnect with RoseOS Intelligence application.

This is not a light QA review.

Your job is to inspect the entire application, compare it against the full requirements below, identify everything missing, incomplete, fake, static, disconnected, broken, or only visually represented, and then implement or fix as much as possible directly in the codebase.

Do not rebuild the app from scratch unless absolutely necessary.

Preserve:
- existing working routes
- existing design system
- existing data models
- existing seeded data
- existing branding
- existing functional components
- existing ServiceConnect dark premium visual style

PRIMARY OBJECTIVE

Verify that ServiceConnect truly functions as:

1. A full BlueFolder replacement
2. A future QuickBooks replacement
3. A ServiceChannel and multi-portal operational hub
4. A GPS-assisted dispatch platform
5. A VoiceConnect-enabled field-service system
6. An inventory, equipment, billing, accounting, document, reporting, and intelligence platform
7. A human-reviewed AI operating system

Do not treat the presence of a menu item, card, modal, or mock data as proof that a feature exists.

A feature only counts as built when:
- the route exists
- the interface exists
- the controls work
- data can be created
- data can be edited
- data persists
- data can be retrieved
- role permissions are enforced where applicable
- workflow states move correctly
- related modules connect to each other
- the feature is not just hardcoded display content
- the feature is clearly labeled as simulated if no real integration exists
